# ARG 重构设计文档：「苔鸣谷」— 回声残响

> **日期**：2026-05-29
> **目标**：将现有 ARG 从无关的黑客叙事重构为《裂月编年：破碎之光》世界观内的支线故事

---

## 1. 故事大纲

### 1.1 背景

**苔鸣谷**：位于白蚀区边缘的瑟帕族聚落，约150只瑟帕，世代研究白蚀裂缝的规律——采集白息样本、观测脉冲周期、通过回声木记录数据。

### 1.2 核心发现

聚落中最年轻的风系灵术师「烬」发现了一个关键规律——白蚀裂缝的脉冲周期与碎月引力波完全同步。她将这个规律命名为「弦律」。这意味着：**大潮灾是可以预测的**。

### 1.3 灭口

黑铁联合体得知此事后，要求苔鸣谷交出所有研究数据，用于军事目的。苔鸣谷拒绝。余火历162年秋，黑铁联合体切断了苔鸣谷的所有补给线，并在一次白蚀脉冲期中人为触发了裂缝扩张。苔鸣谷在一夜之间被白蚀吞噬。

### 1.4 掩盖

官方记录写道：「余火历162年，苔鸣谷因白蚀区自然扩张而覆灭，属天灾，无幸存者。」所有研究数据被列为黑铁联合体最高机密。

### 1.5 反转

烬没有死。她被黑铁联合体抓获，记忆清除后按标准流程处理为「迷失者」，释放至翡翠部落边缘地带。她现在可能就在某个角落里活着，但不记得自己是谁，不记得苔鸣谷，不记得她的研究。

## 2. 四阶段流程设计

### 2.1 阶段一：收集三枚记忆碎片

玩家在网站各处收集烬散落的记录碎片。每获得一枚，都会解锁一段故事文本。宠物作为引导，主动提示信号、指引方向、纠正错误。

#### 碎片一：「白息」(BREATH_WHITE)

- **位置**：首页 (`/`)
- **触发**：在键盘上输入 `M` `O` `S` `S`（苔/Moss，暗指苔鸣谷）
- **反馈**：屏幕泛起白蚀滤镜，中央浮现文字：`「余火历162年·苔鸣谷·白息浓度异常上升……」` + `BREATH_WHITE - 碎片已同步`
- **叙事**：苔鸣谷最后的日常记录，一切还很平静

#### 碎片二：「弦律」(CHORD_PATTERN)

- **位置**：项目页 (`/projects`)
- **触发**：依次点击前3个项目卡片，顺序为 1→2→3（代表三弦：念→灵→渊）
- **反馈**：页面顶端弹出警告框：`「弦律异常·碎月引力波与裂缝脉冲完全同步——烬的研究笔记」` + `CHORD_PATTERN - 碎片已同步`
- **叙事**：烬发现了白蚀的核心规律

#### 碎片三：「求救」(SOS_ECHO)

- **位置**：游戏页 (`/games`)
- **触发**：在非游戏输入状态下，盲打输入 `S` `O` `S`
- **反馈**：系统异常弹窗，低语文字：`「黑铁联合体切断了所有补给线……裂缝在扩张……我们还在……」` + `SOS_ECHO - 碎片已同步`
- **叙事**：烬的求救信号，苔鸣谷正在被毁灭

### 2.2 阶段二：寻找档案入口

- **位置**：文章页 (`/articles`)
- **引导**：宠物提示「烬说她把备份藏在了知识的殿堂……」
- **方法**：找到一篇隐藏的技术文章（或在某篇文章中找到隐藏文字），内容是关于「回声木数据存储技术」的片段
- **隐藏信息**：一段 Base64 编码，解码后得到终端路径（如 `/blackiron-archive`）
- **叙事暗示**：这段文字的语气明显是黑铁联合体的军事档案风格——他们把烬的研究数据保存了下来

### 2.3 阶段三：破解黑铁加密档案

进入终端后，界面是黑铁联合体的军事档案终端风格。

#### 暴力破解（Mastermind 游戏）

- 重新包装：「黑铁联合体军事加密·4位授权码·8次尝试」
- 管理员后门：改为 `echo162`（回声 + 余火历162年）

## 3. 重置机制

通关后，在终端中输入 `reset-arg` 命令可以重置所有 ARG 进度：

- 清除 localStorage 中的所有 ARG 状态
- 重置碎片收集状态
- 重置终端解锁状态
- 重置主题解锁状态
- 重置宠物外观状态
- 终端显示：「ARG 数据已重置。回声木网络信号已清除。」

### 5.2 localStorage 键名

```
arg_keys              → string[]  已收集的碎片ID
arg_terminal_unlocked → boolean   终端是否已解锁
arg_theme_unlocked    → boolean   White Eclipse 主题是否已解锁
arg_theme_active      → boolean   White Eclipse 主题是否激活
arg_pet_dark          → boolean   宠物是否处于暗化状态（purge 奖励）
arg_arg_started       → boolean   ARG 是否已开始
arg_completed         → boolean   ARG 是否已完成
```

### 5.3 宠物 ARG 对话触发条件

- `arg_arg_started === false`：显示 ARG 开始提示
- `arg_completed === true` 且 `arg_pet_dark === true`：显示暗化台词
- 在对应页面且对应碎片未收集：显示提示对话
- 点击宠物且 ARG 进行中：显示当前进度提示
